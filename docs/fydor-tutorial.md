# Fydor Tutorial

Fydor is an open-access language learning app for turning lesson material into structured practice. It focuses on a simple loop: create or import a lesson, check that the lesson data is valid, study it sentence by sentence, review what you forgot, and share useful lessons through Fydor Exchange.

This tutorial is based on the current app structure and the recent commits that added Fydor Exchange, refreshed the sidebar, and expanded the review system.

## 1. The Main Fydor Flow

### Start in Lessons

Open `Lessons` from the sidebar, or go directly to `/lessons/manage`.

This is the main home for saved lessons. From here you can:

- create a new lesson
- select an existing lesson
- open a lesson in the editor
- export a single lesson as JSON
- delete lessons you no longer need

Each lesson is built from sentences. A sentence can include:

- the target-language text
- a translation
- vocabulary words
- grammar patterns
- phrase chunks
- explanations for hover and study hints

Fydor uses sentences as the center of the learning experience because vocabulary and grammar are easier to remember when they stay attached to real usage.

### Build or Import a Lesson

Use the Builder when you want visible fields and controls. Use JSON mode when you want to paste a full lesson generated by an AI assistant or prepared by hand.

The lesson JSON expects:

- `title`
- `language`
- `baseLanguage`
- at least one sentence with `text`

Optional lesson fields include:

- `description`
- `source`
- `level`
- `tags`

Optional sentence annotations include:

- `words`
- `grammar`
- `chunks`

Use `Check` to validate the lesson before saving. Use `Preview` to inspect the sentences, vocabulary, grammar, chunks, duplicates, and validation warnings before committing the lesson to storage.

### Save the Lesson

When the preview looks right, save the lesson. Fydor writes the lesson, sentences, learning items, and sentence-to-item links together so the study pages can reuse the same data.

Existing canonical items are reused instead of duplicated. For example, if the same vocabulary item already exists for the same language, Fydor connects the new sentence to that existing item instead of creating another copy.

## 2. Study Modes

### Flashcards

Open `Flashcards` from the sidebar, or go to `/study/imported-content`.

Flashcards show one sentence at a time. You can reveal layers gradually:

- translation
- word meanings
- grammar notes
- hints

The intended pattern is:

1. Try to understand the sentence without help.
2. Reveal hints or annotations only when you need them.
3. Reveal the translation last.
4. Grade the sentence honestly.

Flashcards can run in regular order or randomized order. The app also inserts checkpoint quiz moments as you move through a lesson.

### Fill Blank

Open `Fill Blank` from the sidebar, or go to `/study/fill-blank`.

This mode turns lesson material into active recall prompts. Instead of only recognizing a sentence, you practice supplying missing words, grammar patterns, or chunks.

Use this when a sentence feels familiar but you still struggle to produce part of it yourself.

### Multiple Choice

Open `Multiple Choice` from the sidebar, or go to `/study/multiple-choice`.

This mode quizzes you from the same saved lesson pool. It is useful for faster recognition practice and for checking whether meanings and annotations are clear.

### Review

Open `Review` from the sidebar, or go to `/review`.

Review is the standalone sentence review queue. It mixes reviewable sentences and lets you mark each card by memory result. Keyboard shortcuts are available while the review deck is active:

- left arrow: not remembered
- right arrow: remembered

The queue gives forgotten sentences higher priority, then unknown sentences, then remembered sentences. Remembered sentences move farther back as their streak increases.

## 3. Fydor Exchange

Fydor Exchange was added as the app's lesson sharing system. Open it from the sidebar under `Create`, or go to `/fydor-exchange`.

Exchange uses Fydor Packs, which are portable `.fydorpack` files containing one or more lessons plus pack metadata.

### Install a Pack

In the `Install Pack` area, you can:

- select a `.fydorpack` file
- paste advanced pack data manually
- preview the pack before installing
- select which lessons inside the pack to install
- decide how duplicates should be handled

The preview checks:

- pack structure
- lesson schema
- lesson count
- sentence count
- author, license, version, tags, language, and level metadata
- duplicate lessons already installed locally

Duplicate modes:

- `Skip existing`: do not install lessons that match an existing title, language, and base language.
- `Replace existing`: update the matching local lesson with the pack version.
- `Keep both`: install a copy with a modified title.

After installation, Fydor refreshes the lesson list and records installed pack metadata in local browser storage so the pack appears in `My Packs`.

### Share a Pack

In the `Share Pack` area, you can:

- select one or more saved lessons
- select all lessons
- clear the current selection
- enter pack metadata
- preview the exported pack
- export selected lessons
- export all lessons

Pack metadata includes:

- title
- description
- author
- organization
- version
- license
- tags

Fydor exports the pack as a JSON-based `.fydorpack` file. The filename is generated from the pack title.

### Manage My Packs

The `My Packs` area shows installed pack records. You can search and filter by:

- pack title
- author
- organization
- tags
- included lesson titles
- language
- level

Each pack record shows its lessons, sentence count, version, license, language pair, level, and tags.

## 4. The Fydor Pack Format

A Fydor Pack has this high-level shape:

```json
{
  "type": "fydor_pack",
  "schemaVersion": 1,
  "id": "example-pack-id",
  "title": "Example Fydor Pack",
  "description": "A short description.",
  "author": {
    "name": "Teacher Name",
    "organization": "School or group"
  },
  "version": "1.0.0",
  "license": "CC BY",
  "language": "zh",
  "baseLanguage": "en",
  "level": "beginner",
  "tags": ["hsk", "daily"],
  "createdAt": "2026-06-26T00:00:00.000Z",
  "updatedAt": "2026-06-26T00:00:00.000Z",
  "lessons": [
    {
      "title": "Example Lesson",
      "language": "zh",
      "baseLanguage": "en",
      "sentences": [
        {
          "text": "Ni hao.",
          "translation": "Hello."
        }
      ]
    }
  ]
}
```

Important rules:

- `type` must be `fydor_pack`.
- `schemaVersion` must currently be `1`.
- a pack must contain at least one lesson.
- each lesson must pass the normal Fydor lesson schema.
- sentence text cannot be empty.
- annotated word and chunk surfaces must appear in the sentence text.
- grammar surfaces must appear in the sentence text when provided.

## 5. Methods Fydor Uses

### Structured Import

Fydor treats lesson content as structured data, not plain notes. The import flow parses JSON, validates required fields, checks surface references, detects duplicates, and builds a preview before saving.

This keeps the same content usable across flashcards, fill-blank prompts, multiple choice, review, and pack export.

### Canonical Reuse

Fydor derives canonical keys from normalized lesson content:

- sentence key: language plus normalized sentence text
- vocabulary key: language plus normalized lemma or surface
- grammar key: language plus normalized pattern
- chunk key: language plus normalized surface

These keys let Fydor reuse existing sentences and learning items instead of scattering duplicates through the database.

### Sentence-Centered Learning

The app anchors vocabulary, grammar, and chunks to full sentences. This supports contextual learning: the learner sees how an item behaves inside a real expression instead of memorizing isolated fragments.

### Progressive Reveal

Flashcards hide support layers by default. The learner can reveal translation, word meanings, grammar, or hints when needed.

This encourages retrieval before recognition. The app gives help, but it does not make help the first step.

### Retrieval Practice

Review, fill-blank, and flashcard grading all ask the learner to pull information from memory. This is the core study method behind Fydor.

### Spaced Review Signals

Review stores sentence-level memory state:

- `unknown`
- `remembered`
- `forgotten`

It also tracks streaks and review timestamps. Forgotten items come back sooner. Remembered items drift later in the queue as streaks increase.

### Interleaving

The standalone review queue can mix sentences from different lessons. This avoids practicing only one narrow pattern at a time and makes recall more realistic.

### Portability

Fydor Packs make lessons portable. A teacher, learner, or community member can export structured lessons and share them without requiring an account, subscription, or hosted marketplace.

## 6. Recommended Workflow

1. Open `Lessons`.
2. Create a lesson in Builder mode or paste lesson JSON.
3. Run `Check`.
4. Use `Preview` to inspect the lesson.
5. Save it.
6. Study it in `Flashcards`.
7. Use `Fill Blank` when you want production practice.
8. Use `Multiple Choice` when you want faster recognition practice.
9. Use `Review` for mixed sentence memory.
10. Use `Fydor Exchange` to export your best lessons as a pack or install someone else's pack.
